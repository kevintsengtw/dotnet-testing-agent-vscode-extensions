# dotnet Testing Agent v0.3.0 安裝指南

> v0.3.0 為功能更新：Claude 模式新增 `.claude/scripts/` 部署類別，支援
> orchestration-claude 的「測試工作流程 Token 用量記錄」功能 —— init 後跑完 orchestrator
> 會自動呈現 Token 用量表（已同步至含 self-location 修正的引擎 v1.1.1）。

## 前置需求

| 項目 | 最低版本 |
| --- | --- |
| VS Code | 1.85.0 |
| Node.js | 18.0.0 |
| 作業系統 | Windows 10/11（macOS / Linux 實驗性支援） |

依使用模式擇一：

| 模式 | 額外需求 |
| --- | --- |
| Copilot 模式 | GitHub Copilot Chat（最新版、已登入） |
| Claude 模式 | Claude Code CLI |

## 下載

從 [Releases](https://github.com/kevintsengtw/dotnet-testing-agent-vscode-extensions/releases/tag/v0.3.0) 下載：

- `dotnet-testing-agent-0.3.0.vsix`

## 安裝 / 更新

本 Extension 以 `.vsix` 發佈、不走 Marketplace，**VS Code 不會自動更新**。從舊版更新時不需先解除安裝，VS Code 會就地以新版覆蓋。

**方法一：Extensions 面板**

1. 開啟 Extensions 面板（`Ctrl+Shift+X`）
2. 點選右上角 `...` → **Install from VSIX...**
3. 選擇下載的 `.vsix` 檔案
4. 依提示 **Reload Window**

**方法二：命令列**

```bash
code --install-extension dotnet-testing-agent-0.3.0.vsix
```

## 主要變更

- **Claude 模式新增 Token 用量功能支援**：新增 `.claude/scripts/` 部署類別，把
  orchestration-claude 的 token-usage 引擎（`token_usage.js`）部署進 workspace。Claude 模式
  `init` 後，orchestrator 跑完測試工作流程會自動在結果最後附上 Token 用量表。
  - `dta.claude.doctor` 新增「`.claude/scripts` 目錄（token-usage 引擎）」診斷項
  - Claude Sidebar 的 Agents 分頁新增「Token 用量引擎」狀態列
- **bundled 引擎同步至 v1.1.1**：含 self-location 修正，解決 IDE 環境因路徑正規化差異
  （`_`→`-`）找不到 session transcript、Token 表不出現的問題。

> 既有 Claude 模式 workspace 升級後，請重新執行 **「初始化 Claude 模式」**（或環境診斷的
> 一鍵修復）以部署新的 token-usage 引擎。

完整變更請見 [CHANGELOG](https://github.com/kevintsengtw/dotnet-testing-vscode-extensions/blob/master/CHANGELOG.md)。
