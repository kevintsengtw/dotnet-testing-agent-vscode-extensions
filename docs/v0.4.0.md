# dotnet Testing Agent v0.4.0 安裝指南

> v0.4.0 為功能更新：Sidebar 最上方新增「版本資訊」面板，一處看完目前安裝版本、
> GitHub 最新 release 版本與 release note，並提示「已更新 extension 但某模式仍以舊版部署」
> 的落差，讓你自行決定是否更新／重新初始化。

## 下載

從 [Releases](https://github.com/kevintsengtw/dotnet-testing-agent-vscode-extensions/releases/tag/v0.4.0) 下載：

- `dotnet-testing-agent-0.4.0.vsix`

## 安裝 / 更新

本 Extension 以 `.vsix` 發佈、不走 Marketplace，**VS Code 不會自動更新**。從舊版更新時不需先解除安裝，VS Code 會就地以新版覆蓋。

**方法一：Extensions 面板**

1. 開啟 Extensions 面板（`Ctrl+Shift+X`）
2. 點選右上角 `...` → **Install from VSIX...**
3. 選擇下載的 `.vsix` 檔案
4. 依提示 **Reload Window**

**方法二：命令列**

```bash
code --install-extension dotnet-testing-agent-0.4.0.vsix
```

## 主要變更

- **新增「版本資訊」面板**：Sidebar（工具包）最上方固定顯示
  - 目前安裝版本（不依賴初始化即可見）
  - GitHub 最新 release 版本、發佈日期與 release note，狀態徽章顯示「已是最新」或「有新版」
  - 「在 GitHub 查看 / 前往下載」連結，更新與否由你決定
  - 部署落差提示：若已更新 extension 但某模式仍以舊版部署，會提示並提供「重新初始化」按鈕

> 啟動時的更新通知（toast）維持不變，與本面板互補。

完整變更請見 [CHANGELOG](https://github.com/kevintsengtw/dotnet-testing-vscode-extensions/blob/master/CHANGELOG.md)。
