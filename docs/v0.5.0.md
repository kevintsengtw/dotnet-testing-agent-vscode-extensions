# dotnet Testing Agent v0.5.0 安裝指南

> v0.5.0 為功能更新：**新增第三種「Codex 模式」**，將 orchestration-codex 的 1+4 測試
> orchestration（unit / TUnit / integration / aspire 四種工作流程）一鍵部署到 workspace
> `.codex/`，供 Codex（VS Code 擴充 / CLI）原生 SpawnAgent 呼叫。同時把內建的
> Copilot / Claude 定義檔一併更新至最新版。

## 前置需求

| 項目 | 最低版本 |
| --- | --- |
| VS Code | 1.85.0 |
| Node.js | 18.0.0 |
| 作業系統 | Windows 10/11（macOS / Linux 實驗性支援） |

依使用模式擇一：

| 模式 | 額外需求 |
| --- | --- |
| Copilot 模式 | GitHub Copilot Chat（最新版、已登入） |
| Claude 模式 | Claude Code CLI |
| Codex 模式 | Codex（VS Code 擴充或 CLI）；integration / aspire 工作流程另需 **Docker** |

## 下載

從 [Releases](https://github.com/kevintsengtw/dotnet-testing-agent-vscode-extensions/releases/tag/v0.5.0) 下載：

- `dotnet-testing-agent-0.5.0.vsix`

## 安裝 / 更新

本 Extension 以 `.vsix` 發佈、不走 Marketplace，**VS Code 不會自動更新**。從舊版更新時不需先解除安裝，VS Code 會就地以新版覆蓋。

**方法一：Extensions 面板**

1. 開啟 Extensions 面板（`Ctrl+Shift+X`）
2. 點選右上角 `...` → **Install from VSIX...**
3. 選擇下載的 `.vsix` 檔案
4. 依提示 **Reload Window**

**方法二：命令列**

```bash
code --install-extension dotnet-testing-agent-0.5.0.vsix
```

## 主要變更

- **新增 Codex 模式**（第三種 AI coding agent 模式，鏡像 Claude 模式）
  - 指令 `初始化 Codex Agent 環境`（`dta.codex.init`）、`Codex 環境診斷`（`dta.codex.doctor`）
  - Sidebar 新增「Codex 模式」面板（3 分頁：總覽 / Skills / Agents）
  - 部署 `.codex/skills/`、`.codex/agents/`（16 個 `.toml`）、`.codex/scripts/`（`run-state.mjs` + `estimate-token-usage.mjs`）、`.codex/config.toml`
  - `config.toml` 採「補差異」合併：保留使用者既有設定、只補上缺少項目
  - doctor 6 項環境檢查（Node.js、Skills/Agents/Config/Scripts、`codex --version`）
  - manifest 為 `manifest-codex.json`；設定 `dta.codex.skillsVersion`
- **內建定義檔更新**：orchestration-codex **v1.0.3**、orchestration-claude **v1.4.0**

## 快速開始（Codex 模式）

1. 命令面板 → `dotnet-testing 測試工作流程: 初始化 Codex Agent 環境`
2. 等待 4 步驟完成（部署 Skills / Agents / Config / Scripts 至 `.codex/`）
3. 在 Codex 中以 `$dotnet-testing-orchestrator-unit`（或 `-tunit` / `-integration` / `-aspire`）喚起對應工作流程

> integration / aspire 工作流程需要 **Docker**。

## 手動驗證

四種工作流程皆已在 Codex VS Code 擴充下實跑通過（核心：測試產生 + build/test + Reviewer）：Unit、TUnit、Integration（Testcontainers）、Aspire（DistributedApplicationTestingBuilder）。

完整變更請見 [CHANGELOG](https://github.com/kevintsengtw/dotnet-testing-vscode-extensions/blob/master/CHANGELOG.md)。
