# dotnet Testing Agent v0.6.0 安裝指南

> v0.6.0 更新三種測試工作流程的內建正式版本，並重構版本檢查、離線處理與安全更新機制。

## 內建正式版本

| 內容 | 版本 |
| --- | --- |
| Shared Skills | v2.4.1 |
| Copilot orchestration | v2.0.1 |
| Claude orchestration | v1.4.0 |
| Codex orchestration | v1.1.0 |

## 前置需求

| 項目 | 最低版本 |
| --- | --- |
| VS Code | 1.85.0 |
| Node.js | 18.0.0 |
| 作業系統 | Windows 10/11；macOS / Linux 依既有支援範圍 |

依使用模式準備 GitHub Copilot Chat、Claude Code 或 Codex。Copilot 模式的 RAG 安裝與索引功能另需 npm／網路或事先準備的離線 model；integration／Aspire 容器測試另需 Docker。

## 下載

從 [v0.6.0 Release](https://github.com/kevintsengtw/dotnet-testing-agent-vscode-extensions/releases/tag/v0.6.0) 下載：

- `dotnet-testing-agent-0.6.0.vsix`

## 安裝或升級

1. 開啟 Extensions 面板（`Ctrl+Shift+X`）。
2. 點選右上角 `...` → **Install from VSIX...**。
3. 選擇 `dotnet-testing-agent-0.6.0.vsix`。
4. 完整結束並重新開啟 VS Code；不要只執行 Reload Window。

命令列安裝：

```bash
code --install-extension dotnet-testing-agent-0.6.0.vsix
```

一般升級不需先解除安裝。既有 Copilot、Claude、Codex 工作區可保留；開啟對應模式面板後，Extension 會檢查工作流程內容版本並提供更新選項。

## 本版重點

- 三模式在 VS Code 啟動、切換到 Extension 模式面板及初始化後檢查正式 Skills／Orchestration Release。
- 背景檢查沒有網路時完全靜默，不會反覆跳出錯誤或警告。
- 使用者主動初始化時可直接使用 VSIX 內建內容；無法檢查最新版時只提示一次。
- 使用者確認更新後若下載或部署失敗，會提示並還原更新前內容。
- Claude／Codex 更新會以 managed-file 清單移除新版已淘汰的受管理檔案，避免長期累積舊 Skills、Agents 與 Scripts。
- 更新保留使用者自訂 skills、agents、scripts、Claude hooks 與 Codex config 設定；v0.5.1 遺留的空 `skill-creator-advanced` 目錄會安全清理。
- 更新失敗後可在同一次 VS Code 執行期間重新檢查並重試；Claude／Codex 更新成功後會自動執行環境診斷。
- Codex v1.1.0 新增 `unit-test-scenarios` 與 validator scripts，Doctor 與面板清單同步更新。

## 更新後確認

- 從 Activity Bar 開啟「dotnet-testing-agent 工具包」。
- 對使用中的模式執行環境診斷。
- 若面板顯示工作流程有新版，可選擇「立即更新」。
- Copilot Skills 更新後建議重建 RAG 索引。

完整變更請見 [CHANGELOG](https://github.com/kevintsengtw/dotnet-testing-vscode-extensions/blob/master/CHANGELOG.md)。
