# dotnet Testing Agent v0.2.0 安裝指南

> 自 v0.2.0 起，Copilot 與 Claude 兩種模式整合為**單一 extension**，安裝一份 `.vsix` 即可同時使用。

## 前置需求

| 項目 | 最低版本 |
| --- | --- |
| VS Code | 1.85.0 |
| Node.js | 18.0.0 |
| 作業系統 | Windows 10/11（macOS / Linux 實驗性支援） |

依使用模式擇一：

| 模式 | 額外需求 |
| --- | --- |
| Copilot 模式 | GitHub Copilot Chat（最新版、已登入） |
| Claude 模式 | Claude Code CLI |

## 下載

從 [Releases](https://github.com/kevintsengtw/dotnet-testing-agent-vscode-extensions/releases/tag/v0.2.0) 下載：

- `dotnet-testing-agent-0.2.0.vsix`

## 安裝

**方法一：Extensions 面板**

1. 開啟 Extensions 面板（`Ctrl+Shift+X`）
2. 點選右上角 `...` → **Install from VSIX...**
3. 選擇下載的 `.vsix` 檔案

**方法二：命令列**

```bash
code --install-extension dotnet-testing-agent-0.2.0.vsix
```

## 使用

安裝後，Activity Bar 會出現「dotnet-testing-agent 工具包」圖示，側邊欄含兩個面板：**Claude 模式** 與 **Copilot 模式**。依需求初始化對應模式。

### Copilot 模式

1. 開啟命令面板（`Ctrl+Shift+P`）
2. 執行：`dotnet-testing 測試工作流程: 初始化 .NET Testing Agent`
3. 選擇安裝模式（線上 / 離線）
4. 等待 6 個步驟完成
5. 開啟 Copilot Chat（`Ctrl+Alt+I`）→ 切換到 **Agent** 模式 → 點選工具圖示（錘子 🔨）→ 確認清單中有 `dotnet-testing-skills`，且 `query_documents`、`list_files` 已勾選

### Claude 模式

1. 開啟命令面板（`Ctrl+Shift+P`）
2. 執行：`dotnet-testing 測試工作流程: 初始化 Claude 模式`
3. 等待 4 個步驟完成（部署 Skills / Agents / Hooks 至 `.claude/`）
4. Claude 模式不需要 mcp-local-rag 或 RAG 索引

## 主要變更

- **新增 Claude 模式**：在單一 Activity Bar 中與 Copilot 模式並存，部署至 `.claude/` 路徑，獨立 manifest
- **整合 Copilot 與 Claude 介面至單一 extension / 單一 `.vsix`**，不再分開發佈
- **發佈 tag 改為 `v*` 系列**，`.vsix` 改名為 `dotnet-testing-agent-<版本>.vsix`（移除 `-copilot` 後綴）
- `bundled/` 目錄重構，共用 Skills 統一存放

完整變更請見 [CHANGELOG](https://github.com/kevintsengtw/dotnet-testing-vscode-extensions/blob/master/CHANGELOG.md)。
